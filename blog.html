Discussion over the scope of var, let, and const
var
  1. The 'var' keyword in JavaScript is used to declare a variable with function scope or, when declared outside of any function, with global scope. This property has both advantages and disadvantages, including the ability to use the variable outside of the block scope where it's declared, which can help avoid variable scope errors or conflicts with same variable names.
  2. If a variable declared with the 'var' keyword is encountered anywhere within a script, it will be hoisted to the top of its containing function or script without its value being initialized at that point i.e it take the value 'undefined' initially.
  3. Variables can be re-declared or reassigned without causing errors
let
  1. "The 'let' keyword is used to declare a variable as a block-scoped variable since 2015, which means it cannot be used outside the scope of the declaration.
  2. Unlike 'var', 'let' variables are not hoisted to the top of the script, but they are hoisted to the top of their containing scope without being initialized if declared anywhere within that scope (e.g., a function or script).
  3. A 'let' variable cannot be re-declared within the same scope, and attempting to do so will result in a 'Cannot redeclare block-scoped variable' error. However, it can be reassigned without any errors.
const
  1. The 'const' keyword is used to declare a variable with block scope since 2015, meaning it cannot be used outside the scope of the declaration.
  2. Unlike 'var', a 'const' variable is not hoisted to the top of the script or function. Instead, it's hoisted to the top of its scope, and it must be initialized when declared. If declared without initialization, it will result in a syntax error.
  3. A 'const' variable cannot be re-declared within the same scope, and it cannot be reassigned after its initial value is set. Attempting to redeclare a 'const' variable will result in a 'Cannot redeclare block-scoped variable' error.
  4. When a 'const' variable holds a primitive value (like a number or string), that value cannot be changed after initialization. However, if a 'const' variable holds a non-primitive value (like an object or array), you can modify the properties or elements of that value without any error, as long as you don't attempt to reassign the variable itself.
the use cases of null and undefined
null
  1. The type of a variable assigned a value of null is 'object,' which is a non-primitive type in JavaScript.
  2. We can assign the value null to any variable when you intentionally want to indicate the absence of a value.
  3. Null is often used to reset objects or variables, effectively clearing their value or reference.
  4. null & undefined are same in value but deference in type
  5. We can set an individual element within an array to null when there is a lack of a specific value for that element.
  6. Null is commonly used in conditional statements to check if a variable or value is absent or uninitialized.
undefined
  1. The type of an undefined variable is 'undefined,' which is a primitive type in JavaScript.
  2. Variables are assigned the value 'undefined' if they are declared but not initialized. This is the default behavior in JavaScript.
  3. Undefined is sometimes used to indicate that a variable has no assigned value or to reset variables to their uninitialized state.
  4. In JavaScript, function parameters that are not provided with values during a function call are automatically set to 'undefined' by default.
  5. A function that doesn't explicitly return a value will return 'undefined' by default.
  6. 'Undefined' is commonly used in conditional statements to check if a variable has no value or is uninitialized.
  7. The array or object will return undefined for missing index or key
The meaning of REST API
  1. The full form of REST API is "REpresentational State Transfer Application Programming Interface."
  2. REST APIs are characterized by being stateless, which means that each interaction or request between a client and a server should contain all the information necessary for the server to understand and process it.
  3. REST APIs use HTTP methods for performing operations on resources.
  4. They have a uniform interface, meaning the API follows a consistent and standardized set of rules and conventions for interacting with resources.
  5. REST APIs use standard HTTP methods such as GET, POST, PUT, DELETE, etc., to perform CRUD (Create, Read, Update, Delete) operations.
  6. In REST, everything is considered as a resource.
  7. Resources in REST are identified by URIs (Uniform Resource Identifiers).
  8. RESTful messages are self-descriptive, meaning the messages themselves contain information about how to process them.
  9. APIs in a RESTful architecture can be navigated using hyperlinks.
