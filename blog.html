Discussion over the scope of var, let, and const
var
  1. The 'var' keyword is used to declare a variable as a global variable since early js, meaning it can be used outside the scope of the declaration. This property is cause of various pros and cons, such as avoiding variable scope errors or conflicts with variable names.
  2. It would be hoisted to top of the script without initializations if declared anywhere in the script
  3. It can be re-declared or reassigned without any errors
let
  1. The 'let' keyword is used to declare a variable as a local variable since 2015, meaning it can't be used outside the scope of the declaration.
  2. It wouldn't be hoisted to top of the script but top of the scope without initializations if declared anywhere in the scope i.e function or script or document.
  3. It can't be re-declared for 'Cannot redeclare block-scope variable' but can reassigned without any errors
const
  1. The 'let' keyword is used to declare a variable as a local variable since 2015, meaning it can't be used outside the scope of the declaration.
  2. It wouldn't be hoisted to top of the script but top of the scope without initializations if declared anywhere in the scope i.e function or script or document.
  3. It can't be re-declared for 'Cannot redeclare block-scope variable' nor reassigned.
  4. It contains only unchanged value. But in non-primitive value, we can change the value of an entry without any error.
the use cases of null and undefined
null
  1. The type of a null valued variable is object - a non-primitive type
  2. Any variables can assigned to null if one don't want to set any value intensionally
  3. To reset objects / variables, sometimes we use to set null to the objects / variables
  4. Any parameters in a function without passing the values can set to null by default
  5. One can set null to an array element for lack of the element value
  6. null often used in conditional statements
undefined
  1. The type of a undefined variable is undefined - a primitive type
  2. Any variables is assigned to undefined if not initialized by default
  3. To reset variables, sometimes we use set undefined to the variable
  4. Any parameters in a function without passing the values are set to undefined by default
  5. Any function that return nothing is return undefined by default
  6. undefined often used in conditional statements
The meaning of REST API
  1. Here REST API full form is REpresentational State Transfer Application Programming Interface.
  2. which characterized by being stateless meaning [each interaction or request between a client and a server should contain all the information necessary for the server to understand and process]
  3. which using HTTP methods for operations on resources
  4. It has uniform interface [i.e. the API follows a consistent and standardized set of rules and conventions for interacting with resources]
  5. It uses standard HTTP methods [GET, POST, PUT, DELETE, etc.] to perform the CRUD
  6. everything considered as resources
  7. the Resources are identified by URIs (Uniform Resource Identifiers).
  8. RESTful messages are self-descriptive
  9. APIs are navigate with hyperlinks
